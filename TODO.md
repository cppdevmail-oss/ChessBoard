# Onboarding / Tutorial TODO

Этот раздел добавлен как подробная handoff-спецификация для другого разработчика или другого Codex-агента.
Старое содержимое файла выше не менять и не удалять без отдельного решения.

## 1. Цель

Нужно реализовать обучение базовому использованию приложения.

Первый scope:
- только одно обучение
- только сценарий ручного ввода партии
- обучение работает только в `SimpleView`

Обучение должно провести пользователя через первый рабочий flow:
1. пользователь заходит в приложение
2. пользователь создаёт партию вручную
3. пользователь запускает тренировку по этой партии
4. пользователь проходит тренировку
5. приложение фиксирует завершение обучения

Важно:
- это не справка и не просто подсказки
- это guided flow с жёсткими ограничениями по доступным экранам и действиям
- пользователь должен иметь возможность остановить обучение, если что-то пошло не так

## 2. Что уже обсуждено и согласовано

Согласованные правила:

1. Обучение запускается только в `SimpleView`.
2. Если пользователь входит в обучение, приложение принудительно включает `SimpleView`.
3. После завершения или остановки обучения автоматически возвращать прошлое значение `SimpleView` не нужно.
4. Первый tutorial только один:
   - ручной ввод партии
5. Сигнал, что tutorial нужно предлагать:
   - в базе нет партий
   - и нет статистики
6. На главном экране `SimpleView` нужно добавить иконку `?`.
7. Если tutorial ещё не идёт:
   - по `?` предлагается начать обучение
8. Если tutorial уже активен:
   - по `?` показывается подсказка, что делать на текущем шаге
9. Во время обучения в `TopBar` должна быть красная кнопка остановки tutorial.
10. Прогресс tutorial нужно хранить в отдельной таблице базы данных.

## 3. Какой именно tutorial делаем сейчас

Сейчас делаем только один сценарий:
- `Manual first flow`

То есть пользователь:
- не вставляет PGN
- не импортирует файл
- именно руками вводит партию

Всё остальное пока не реализуем:
- обучение через PGN text
- обучение через PGN file
- выбор между несколькими tutorial-сценариями

Это важно не раздувать.

## 4. Этапы tutorial

Нужно использовать именно такие этапы:

1. `START`
2. `GAME_CREATED`
3. `TRAINING_CREATED`
4. `TRAINING_STARTED`
5. `TRAINING_COMPLETED`

### 4.1 `START`

Смысл этапа:
- пользователь ещё ничего не создал
- он находится на главном экране
- ему нужно нажать только кнопку создания партии

Ограничения:
- доступен только `Home` в `SimpleView`
- разрешена только кнопка `Create game`
- всё лишнее либо скрыто, либо disabled

Переход дальше:
- пользователь нажал кнопку создания партии
- открылся экран создания партии

### 4.2 `GAME_CREATED`

Смысл этапа:
- пользователь должен вручную создать и сохранить партию

На экране создания партии в tutorial-режиме:
- доступна доска
- доступен выбор стороны
- доступна кнопка `Save`

Недоступно:
- PGN text block
- import from file
- другие способы наполнения партии

Важно:
- если при сохранении сработал duplicate и приложение вернуло ID уже существующей партии, это допустимо
- tutorial должен запомнить этот ID как `trackedGameId`
- если сохранение дало несколько партий, запоминается первый ID

Этап считается пройденным, когда:
- tutorial сохранил `trackedGameId`
- и партия с этим ID реально существует в базе

### 4.3 `TRAINING_CREATED`

Смысл этапа:
- пользователь не создаёт тренировку в базе
- пользователь создаёт runtime-training из tutorial-партии

Текущее решение:
- тренировка создаётся динамически
- не сохраняется в БД
- хранится в `runtimecontext`

На этом этапе:
- пользователь возвращается на `Home`
- доступна только кнопка `Smart Training`

На экране `SmartTrainingScreen` нужно добавить кнопку:
- `Runtime Training`

Поведение кнопки:
- в обычном режиме:
  - создаёт runtime training по текущей обычной логике smart training
- в tutorial-режиме:
  - создаёт runtime training из одной единственной партии `trackedGameId`

Этап считается пройденным, когда:
- runtime training session для tutorial реально создана из `trackedGameId`

### 4.4 `TRAINING_STARTED`

Смысл этапа:
- пользователь реально вошёл в экран тренировки, созданной на предыдущем шаге

Этап считается пройденным, когда:
- экран tutorial runtime training реально открыт
- текущая session действительно использует tutorial game

Недостаточно:
- просто нажать кнопку
- просто попытаться открыть тренировку

### 4.5 `TRAINING_COMPLETED`

Смысл этапа:
- пользователь завершил tutorial training
- по партии записана статистика

Этап считается пройденным, когда:
- training flow завершён
- статистика по tutorial game записана

После этого:
- tutorial помечается completed
- ограничения tutorial снимаются

## 13. Критерии готовности первого tutorial

Можно считать первый этап реализации успешным, если:

1. На `Home` в `SimpleView` есть `?`.
2. При отсутствии партий и статистики по `?` можно стартовать tutorial.
3. При старте tutorial включается `SimpleView`.
4. На `START` доступна только кнопка `Create game`.
5. На `CreateOpening` в tutorial доступны только ручной ввод, выбор стороны и save.
6. После save tutorial запоминает `trackedGameId`.
7. После этого на `Home` доступен только `Smart Training`.
8. В `SmartTrainingScreen` можно создать tutorial runtime training по одной партии.
9. Старт и завершение tutorial training переводят tutorial по stage.
10. По завершении tutorial получает статус `COMPLETED`.
11. Во время tutorial есть `?` и stop.
12. После stop приложение не зависает в заблокированном состоянии.
