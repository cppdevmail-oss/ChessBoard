# 📘 Документация: система тренировки дебютов

## 1. Общая идея

Система тренировки состоит из двух уровней:

1. **Шаблон тренировки (Training Template)**  
   — статический набор партий с весами

2. **Тренировка (Training)**  
   — “живая” копия шаблона, в которой веса изменяются по мере прохождения

---

## 2. Шаблон тренировки

### 📦 Назначение

Шаблон — это **исходный набор вариантов**, который пользователь хочет тренировать.

Он не изменяется во время тренировки и может использоваться многократно.

---

### 🗄 Структура

Таблица: `training_templates`

```
id        — уникальный ID
name      — название шаблона
gamesJson — JSON со списком партий и их весами
```

---

### 📄 Формат `gamesJson`

```json
[
  { "gameId": 1, "weight": 3 },
  { "gameId": 5, "weight": 1 }
]
```

- `gameId` — ссылка на партию
- `weight` — сколько раз этот вариант должен встретиться в тренировке

---

### ⚙️ Операции

- добавить партию (или изменить вес)
- удалить партию
- получить список партий

---

## 3. Тренировка

### 📦 Назначение

Тренировка — это **копия шаблона**, которая изменяется в процессе прохождения.

👉 ключевая идея:
> шаблон — неизменяемый  
> тренировка — изменяемая

---

### 🗄 Структура

Таблица: `trainings`

```
id        — уникальный ID
name      — имя (копируется из шаблона)
gamesJson — копия JSON из шаблона
```

---

## 4. Создание тренировки

Вход: `templateId`

### 🔄 Процесс:

1. Получаем шаблон
2. Копируем:
    - `name`
    - `gamesJson`
3. Создаём запись в `trainings`

👉 Важно:
- данные копируются полностью
- шаблон не изменяется

---

## 5. Логика прохождения тренировки

Тренировка работает как **мультимножество партий**:

- каждая партия имеет вес
- вес = сколько раз её нужно пройти

---

## 6. Уменьшение веса

### 📥 Метод:
`decreaseWeight(trainingId, gameId)`

### 🔄 Поведение:

1. Найти партию в `gamesJson`
2. Уменьшить вес на 1

### 📌 Варианты:

#### ✅ Вес > 1
```
3 → 2
```

→ просто обновляем

#### ❗ Вес = 1
```
1 → 0
```

→ партия удаляется из списка

#### 💀 После удаления список пуст

```json
[]
```

→ тренировка удаляется полностью

---

## 7. Увеличение веса

### 📥 Метод:
`increaseWeight(trainingId, gameId)`

### 🔄 Поведение:

1. Найти партию
2. Увеличить вес:

```
2 → 3
```

#### ❗ Если партия не найдена

→ ничего не происходит (можно изменить в будущем)

---

## 8. Жизненный цикл тренировки

```
Шаблон → Создание → Тренировка → Постепенное уменьшение → Удаление
```

### Пример:

```json
[
  { "gameId": 1, "weight": 2 },
  { "gameId": 2, "weight": 1 }
]
```

#### Шаги:

1. сыграли gameId=1 → weight=1
2. сыграли gameId=2 → удалён
3. сыграли gameId=1 → удалён

👉 список пуст → тренировка удалена

---

## 9. Особенности реализации

### ✔ Плюсы

- простая структура (1 поле JSON)
- не нужны сложные JOIN
- легко копировать шаблон → тренировку
- гибкость (можно менять формат JSON)

### ⚠️ Ограничения

- нет FK (возможны “битые” gameId)
- нельзя делать SQL-запросы по играм
- вся логика в коде

---

## 10. Концептуальная модель

```
Template (immutable)
        ↓
   copy JSON
        ↓
Training (mutable)
        ↓
weights ↓
        ↓
empty → delete
```

---

## 🚀 Возможное развитие

- выбор следующей партии по весу
- случайный выбор (weighted random)
- статистика (успех/ошибки)
- интервальные повторения (spaced repetition)
